---
title: "Resource Scheduling"
description: "API reference for ResourceAwareExecutor, ResourcePool, ResourceRequirements, and resource tracking — GPU, CPU, and memory-aware scheduling."
---

The resource scheduling module extends the standard executor with capacity-aware
scheduling. Nodes declare their resource requirements (GPU, CPU, memory, or
custom resources), and the executor only dispatches a node when the resource pool
has sufficient capacity. This prevents oversubscription and enables scheduling of
heterogeneous workloads.

See the [Resource Scheduling](/guide/execution-strategies/resource-scheduling) guide for usage
patterns and capacity planning.

---

## ResourceAwareExecutor

<ApiSignature name="ResourceAwareExecutor" signature={`class ResourceAwareExecutor(
    dag: DAG,
    resource_pool: ResourcePool,
    requirements: dict[str, ResourceRequirements],
    costs: dict[str, float] | None = None,
    callbacks: ExecutionCallbacks | None = None,
    fail_fast: bool = True,
    enable_tracing: bool = False,
)`} />

A synchronous executor that checks resource availability before dispatching each
node. When a node cannot be scheduled due to insufficient resources, it blocks
until resources are released by completed nodes.

<ParamTable params={[
  {name: "dag", type: "DAG", description: "The DAG whose topology drives execution order."},
  {name: "resource_pool", type: "ResourcePool", description: "The pool of available resources."},
  {name: "requirements", type: "dict[str, ResourceRequirements]", description: "Map of node names to their resource requirements."},
  {name: "costs", type: "dict[str, float] | None", default: "None", description: "Per-node cost estimates for scheduling priority."},
  {name: "callbacks", type: "ExecutionCallbacks | None", default: "None", description: "Optional lifecycle callbacks. Includes on_resource_acquired and on_resource_released."},
  {name: "fail_fast", type: "bool", default: "True", description: "If True, skip downstream nodes when any node fails."},
  {name: "enable_tracing", type: "bool", default: "False", description: "If True, record a Chrome-compatible execution trace."},
]} />

### execute

<ApiSignature name="ResourceAwareExecutor.execute" signature={`def execute(
    tasks: dict[str, Callable],
) -> ExecutionResult`} />

Execute tasks with resource-aware scheduling.

<ParamTable params={[
  {name: "tasks", type: "dict[str, Callable]", description: "Map of node names to callable tasks."},
]} />

**Returns:** [ExecutionResult](/api/execution/execution)

```python
import dagron

dag = (
    dagron.DAG.builder()
    .add_node("preprocess")
    .add_node("train_model_a")
    .add_node("train_model_b")
    .add_node("evaluate")
    .add_edge("preprocess", "train_model_a")
    .add_edge("preprocess", "train_model_b")
    .add_edge("train_model_a", "evaluate")
    .add_edge("train_model_b", "evaluate")
    .build()
)

pool = dagron.ResourcePool(capacities={"gpu": 2, "cpu": 8, "memory_mb": 16000})

requirements = {
    "preprocess": dagron.ResourceRequirements(resources={"cpu": 2, "memory_mb": 2000}),
    "train_model_a": dagron.ResourceRequirements(resources={"gpu": 1, "cpu": 4, "memory_mb": 8000}),
    "train_model_b": dagron.ResourceRequirements(resources={"gpu": 1, "cpu": 4, "memory_mb": 8000}),
    "evaluate": dagron.ResourceRequirements(resources={"cpu": 2, "memory_mb": 4000}),
}

executor = dagron.ResourceAwareExecutor(dag, pool, requirements)
result = executor.execute({
    "preprocess": lambda: "data ready",
    "train_model_a": lambda: "model A trained",
    "train_model_b": lambda: "model B trained",
    "evaluate": lambda: "evaluation complete",
})

print(result.succeeded)  # 4
```

### execute (async)

<ApiSignature name="AsyncResourceAwareExecutor.execute" signature={`async def execute(
    tasks: dict[str, Callable[..., Awaitable]],
) -> ExecutionResult`} />

Execute async tasks with resource-aware scheduling.

```python
import asyncio
import dagron

async def main():
    executor = dagron.AsyncResourceAwareExecutor(dag, pool, requirements)
    result = await executor.execute({
        "preprocess": lambda: preprocess_async(),
        "train_model_a": lambda: train_async("model_a"),
        "train_model_b": lambda: train_async("model_b"),
        "evaluate": lambda: evaluate_async(),
    })
    print(result.succeeded)

asyncio.run(main())
```

## ResourcePool

<ApiSignature name="ResourcePool" signature={`class ResourcePool(
    capacities: dict[str, float],
)`} />

A pool of named resources with finite capacities. Resources are acquired before
a node runs and released after it completes (or fails).

<ParamTable params={[
  {name: "capacities", type: "dict[str, float]", description: "Map of resource names to their total capacities."},
]} />

### can_satisfy

<ApiSignature name="ResourcePool.can_satisfy" signature={`def can_satisfy(
    requirements: ResourceRequirements,
) -> bool`} />

Return `True` if the pool's total capacity can satisfy the given requirements
(ignoring current allocation). Useful for validating requirements before
execution.

### try_acquire

<ApiSignature name="ResourcePool.try_acquire" signature={`def try_acquire(
    requirements: ResourceRequirements,
) -> bool`} />

Attempt to acquire resources without blocking. Returns `True` if successful,
`False` if insufficient resources are currently available.

### acquire

<ApiSignature name="ResourcePool.acquire" signature={`def acquire(
    requirements: ResourceRequirements,
    timeout: float | None = None,
) -> bool`} />

Acquire resources, blocking until they become available or the timeout expires.
Returns `True` on success, `False` on timeout.

<ParamTable params={[
  {name: "requirements", type: "ResourceRequirements", description: "The resources to acquire."},
  {name: "timeout", type: "float | None", default: "None", description: "Maximum time to wait in seconds. None means wait indefinitely."},
]} />

### release

<ApiSignature name="ResourcePool.release" signature={`def release(
    requirements: ResourceRequirements,
) -> None`} />

Release previously acquired resources back to the pool.

### Properties

<ApiSignature name="ResourcePool.capacities" signature={`@property
def capacities() -> dict[str, float]`} />

The total capacity of each resource.

<ApiSignature name="ResourcePool.available" signature={`@property
def available() -> dict[str, float]`} />

The currently available (unallocated) amount of each resource.

<ApiSignature name="ResourcePool.allocated" signature={`@property
def allocated() -> dict[str, float]`} />

The currently allocated amount of each resource.

<ApiSignature name="ResourcePool.timeline" signature={`@property
def timeline() -> ResourceTimeline`} />

A timeline of resource allocation events for visualization and debugging.

```python
pool = dagron.ResourcePool(capacities={"gpu": 4, "cpu": 16, "memory_mb": 32000})

print(pool.capacities)   # {"gpu": 4, "cpu": 16, "memory_mb": 32000}
print(pool.available)    # {"gpu": 4, "cpu": 16, "memory_mb": 32000}

req = dagron.ResourceRequirements(resources={"gpu": 2})
pool.acquire(req)
print(pool.available)    # {"gpu": 2, "cpu": 16, "memory_mb": 32000}
print(pool.allocated)    # {"gpu": 2, "cpu": 0, "memory_mb": 0}

pool.release(req)
print(pool.available)    # {"gpu": 4, "cpu": 16, "memory_mb": 32000}
```

## ResourceRequirements

<ApiSignature name="ResourceRequirements" signature={`class ResourceRequirements(
    resources: dict[str, float],
)`} />

A set of resource requirements for a single node.

<ParamTable params={[
  {name: "resources", type: "dict[str, float]", description: "Map of resource names to required amounts."},
]} />

### Convenience Constructors

<ApiSignature name="ResourceRequirements.gpu" signature={`@classmethod
def gpu(n: float) -> ResourceRequirements`} />

Create a requirement for `n` GPUs.

<ApiSignature name="ResourceRequirements.cpu" signature={`@classmethod
def cpu(n: float) -> ResourceRequirements`} />

Create a requirement for `n` CPU cores.

<ApiSignature name="ResourceRequirements.memory" signature={`@classmethod
def memory(mb: float) -> ResourceRequirements`} />

Create a requirement for `mb` megabytes of memory.

### fits

<ApiSignature name="ResourceRequirements.fits" signature={`def fits(
    available: dict[str, float],
) -> bool`} />

Return `True` if the given available resources can satisfy this requirement.

<ParamTable params={[
  {name: "available", type: "dict[str, float]", description: "Map of resource names to available amounts."},
]} />

```python
# Explicit construction
req = dagron.ResourceRequirements(resources={"gpu": 2, "cpu": 4, "memory_mb": 8000})

# Convenience constructors
gpu_req = dagron.ResourceRequirements.gpu(1)
cpu_req = dagron.ResourceRequirements.cpu(4)
mem_req = dagron.ResourceRequirements.memory(4096)

# Check fit
print(req.fits({"gpu": 4, "cpu": 16, "memory_mb": 32000}))  # True
print(req.fits({"gpu": 1, "cpu": 16, "memory_mb": 32000}))  # False (needs 2 GPUs)
```

## ResourceSnapshot

<ApiSignature name="ResourceSnapshot" signature={`class ResourceSnapshot(
    timestamp: float,
    allocated: dict[str, float],
    available: dict[str, float],
    node_name: str | None,
    event: str,
)`} />

A point-in-time snapshot of resource allocation. Recorded by the
`ResourceTimeline`.

| Property | Type | Description |
|----------|------|-------------|
| `timestamp` | `float` | Unix timestamp of the snapshot. |
| `allocated` | `dict[str, float]` | Allocated resources at this point. |
| `available` | `dict[str, float]` | Available resources at this point. |
| `node_name` | `str | None` | The node that triggered this allocation event, if any. |
| `event` | `str` | Event type: `"acquire"` or `"release"`. |

## ResourceTimeline

<ApiSignature name="ResourceTimeline" signature={`class ResourceTimeline`} />

A recorded timeline of resource allocation events. Access via
`ResourcePool.timeline`.

### record

<ApiSignature name="ResourceTimeline.record" signature={`def record(
    snapshot: ResourceSnapshot,
) -> None`} />

Manually record a snapshot (typically done automatically by the pool).

### snapshots

<ApiSignature name="ResourceTimeline.snapshots" signature={`@property
def snapshots() -> list[ResourceSnapshot]`} />

All recorded snapshots in chronological order.

### peak_utilization

<ApiSignature name="ResourceTimeline.peak_utilization" signature={`def peak_utilization() -> dict[str, float]`} />

Return the peak utilization (as a fraction 0.0-1.0) for each resource across
the entire timeline.

```python
result = executor.execute(tasks)

timeline = pool.timeline
print(f"Snapshots: {len(timeline.snapshots)}")
print(f"Peak utilization: {timeline.peak_utilization()}")
# {"gpu": 1.0, "cpu": 0.75, "memory_mb": 0.5}

for snap in timeline.snapshots:
    print(f"  t={snap.timestamp:.3f} {snap.event} {snap.node_name}: gpu={snap.allocated.get('gpu', 0)}")
```
